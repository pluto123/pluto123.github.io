  • getDerivedStateFromProps
    此方法在 render 之前被呼叫，所以很適合將元件初始的 props 設定到 state 物件。

但是須注意，當元件更新時，也會在渲染前呼叫 getDerivedStateFromProps()，以底下範例就算是點擊按紐觸發更新，但是因為渲染前呼叫getDerivedStateFromProps()，所以 state 物件的 name (Eric)，又被改回成 Props 的 initName (Amy)

  • shouldComponentUpdate
    shouldComponentUpdate() 返回一個布林值，React 依據該值決定是否應該要更新元件，預設值為 true。

  • getSnapshotBeforeUpdate
    可以讓使用者可取得改變前的 props 和 state 物件的值

  • React Memo
    問題 :
    假設頁面中有父子組件，當父組件發生事件改變時，就算子組件並未受影響，但也連帶會讓子組件進行重新渲染。
    index.js App.js Todos.js 此時就算按父組件的按鈕，其影響只是父組件的顯示，子組件並不會改變，但看 console 一直輸出 “todos render”，則實際上 React 也會去重新渲染子組件。 解法 :
    使用 memo 將子組件包裹
    Todos.js 按父組件按鈕，此時並不會讓子組件重新渲染了。 按子組件按鈕時，才會重新渲染。
