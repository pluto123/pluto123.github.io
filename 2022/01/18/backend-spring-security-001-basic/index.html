Spring Security Notes - 001 基礎概念

Posted on 2022-01-18 Edited on 2022-01-19 In Spring Security
Symbols count in article: 4.4k Reading time ≈ 4 mins.

Spring 自動配置項目

當專案一引入底下依賴，Spring Security 就會自動配置一些東西

  1. 當專案一啟動時，可以在 Console 中看到底下文字，表示 Spring Security 自動幫我們建立一個帳號為 user，密碼為 bbd1ed75-49a8-4284-9bef-297fd4378d31，此密碼為動態產生的，每次啟動都會不同。

  2. 根據配置，Spring Sercuity 會配置不同的 Filter Chains，底下只是其中一個範例

  3. 預設所有請求都必須經過認證，且會產生一個頁面 (/login)，讓使用者輸入帳號與密碼

Spring Security 三大設定模塊

AuthenticationManagerBuilder

主要可配置

  • AuthenticationProvider，提供驗證服務
    先建立一個 CustomAuthenticationProvider 類別實作 AuthenticationProvider 介面 透過繼承 WebSecurityConfigurerAdapter 的類別中設定
  • UserDetailsService，提供使用者詳情服務
    先建立一個 CustomUserDetailsService 類別實作 UserDetailsService 介面 透過繼承 WebSecurityConfigurerAdapter 的類別中設定

WebSecurity

此設定一般都不太會去做修改，以免危害到系統安全

  • HttpFirewall 配置
    先建立一個 Bean，透過繼承 WebSecurityConfigurerAdapter 的類別中設定

    注意 : 雖然提供修改方式解除一些限制，一般都不建議做任何的改變，以免影響系統安全

  • debug 配置
    透過繼承 WebSecurityConfigurerAdapter 的類別中設定是否要輸出除錯訊息

  • 全局請求忽略規則配置
    底下設定會忽略檢查存取 /static/ 下的所有請求

  • 全局 SecurityFilterChain 配置

  • 權限評估器
    默認是 DefaultWebInvocationPrivilegeEvaluator

  • 表達式處理器
    默認是 DefaultWebSecurityExpressionHandler

HttpSecurity

配置各種驗證相關的設定，常用到的配置項

  • csrf()
    支援 CSRF 支持
  • authorizeRequests()
    配置路徑訪問是否需要通過認證與所需要的權限
  • formLogin()
    開啟使用表單的身份驗證，如果沒有透過 loginPage(URL) 設定登入頁面的路徑，則預設路徑為 /login
  • rememberMe()
    開啟 Remember Me 驗證
  • httpBasic()
    配置 basic 登錄

WebSecurityConfigurerAdapter

以上設定都是通過繼承 WebSecurityConfigurerAdapter 後覆寫底下三種方法來做設定
