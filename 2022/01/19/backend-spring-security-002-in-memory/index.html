Spring Security Notes - 002 使用者帳密訊息儲存於 Memory (僅限測試時用)

Posted on 2022-01-19 Edited on 2022-01-24 In Spring Security
Symbols count in article: 7.5k Reading time ≈ 7 mins.

實作 UserDetailsService 介面

src/main/java/tw/com/pluto/service/MyUserDetailsService.java

配置 UserDetailsService

繼承 WebSecurityConfigurerAdapter 並在 configure(AuthenticationManagerBuilder auth) 中設定
src/main/java/tw/com/pluto/config/MyWebSecurityConfig.java

設定密碼加密

  1. 加入加密的 bean 並使用加密函數將密碼加密
    src/main/java/tw/com/pluto/service/MyUserDetailsService.java
  2. 將加密函數設定給驗證模組，以便在使用者輸入明碼時，將明碼經過加密函數加密後與伺服器上的密碼比對
    src/main/java/tw/com/pluto/config/MyWebSecurityConfig.java 目前 Spring Security 提供的加密函數，預設是使用 BCryptPasswordEncoder()

權限設定

路徑權限的設定必須繼承 WebSecurityConfigurerAdapter 後，覆寫 configure(HttpSecurity http) 設定
src/main/java/tw/com/pluto/config/MyWebSecurityConfig.java

上面的設定是 URL = / 的話，不用登入就可以取得頁面

URL = /user 的話，必須具有角色 USER 的身分，因為目前使用者 user123 具有該角色身分，所以可順利取得

URL = /user 的話，必須具有角色 USER 的身分，因為目前使用者 user123 不具有該角色身分，所以被禁止

Note:

  1. 如果有設定路徑權限的話就必須要明確設定 formLogin() 才可以
  2. mvcMatchers() 和 antMatcher() 的差異，基本上 mvcMatchers() 會考慮比較完備，詳細請參考 https://stackoverflow.com/questions/50536292/difference-between-antmatcher-and-mvcmatcher

階層權限設定

Spring Security 的角色權限一般是不具有別的角色的權限, 如有兩個角色 ADMIN 和 USER，具有 ADMIN 角色的使用者只有 ADMIN 權限，沒辦法儲存需 USER 權限的資源，但是一般應用的情景會讓 ADMIN 也具有 USER 的權限。

新增一個 RoleHierarchy Bean
src/main/java/tw/com/pluto/config/MyWebSecurityConfig.java
