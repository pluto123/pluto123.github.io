實作 UserDetailsService 介面

當 Spring Security 由網頁中取得 username 後，會呼叫 UserDetailsService 中的 UserDetails loadUserByUsername(String username) 方法取得資料庫(或其他方式)中的帳密資訊，將其封裝成 UserDetails 物件，並返回給 Spring Security 做比對密碼的動作與授權。

實作 UserDetails 介面

因為在 UserDetailsService 的 loadUserByUsername() 必須返回一個 UserDetails 物件，所以我們必須實作 UserDetails 介面以提供相關資訊。

因為 amy 具有 ADMIN 角色，所以可以進入 /admin

階層權限設定
