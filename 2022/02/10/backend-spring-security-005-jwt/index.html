事前準備

準備一個前後端分離的專案，以便後續將使用 Session 機制，改寫成使用 JWT 方式
https://github.com/pluto123/spring-security-exercise/tree/jwt

改寫思維

整個流程大概是寫一個自定義過濾器用來捕抓 TOKEN 進行驗證，如果沒有 TOKEN 則會進入帳密驗證程序，驗證成功則返回一個 TOKEN 給使用者，使用者下次請求時附上 TOKEN 就可以不需要帳密驗證，直接返回資源。

JWT 設定

secret_key 的值要參考 HS512 演算法的規定，可參考 https://8gwifi.org/jwkfunctions.jsp

自定義 Filter
