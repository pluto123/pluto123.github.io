  • bubbling and capturing
    參考
    bubbling : 事件傳遞方向是由內至外
    capturing : 事件傳遞方式是由外至內

  • Browser Object Model (BOM)
    BOM 是 JavaScript 與瀏覽器對話的橋樑，但是並沒有一個標準，依據每個瀏覽器而異。

  1. window
    瀏覽器視窗物件，但是其視窗範圍不包含 toolbar and scrollbar。
    ex : let width = window.innerWidth; // 取得瀏覽器視窗寬度 (單位 : pixel)
  2. window.screen
    使用者螢幕資訊，可以不需要 window 前綴，使用 screen 即可
    ex : let width = screen.width; // 取得使用者螢幕寬度 (單位 : pixel)
  3. window.location
    可用於取得當前頁面地址 (URL)或是將瀏覽器重定向。
    ex : let url = window.location.href; // 取得當前頁面 URL
  4. window.history
    瀏覽器的歷史紀錄
    ex : history.back(); // 返回上一頁
  5. window.navigator
    使用者的瀏覽器資訊
  6. window.alert(“sometext”)
    跳出一些訊息讓使用者知道
  7. window.confirm(“sometext”)
    跳出選項讓使用者確認
  8. window.prompt(“sometext”,”defaultText”)
    跳出視窗讓使用者輸入一些訊息
  9. window.setTimeout(function, milliseconds)
    可設定定時器經過 milliseconds 後呼叫 function
  10. window.clearTimeout(timeoutVariable)
    清除定時器
  11. window.setInterval(function, milliseconds)
    可設定瀏覽器每隔 milliseconds 後就呼叫 function
  12. window.clearInterval(timerVariable)
    清除間隔呼叫
  13. document.cookie
    可設定、取出及清除 cookie

Posted on 2022-06-30 In React
Symbols count in article: 789 Reading time ≈ 1 mins.

參考出處
自從 React 16.8 版之後，推出了 Hooks，讓 Functional Component 也具有 Class Component 一樣的功能。
底下是一個小例子:

Hooks 出現，目前不是為了取代 Class Component 的，所以他可以使用在任何版本為 16.8 以上的專案，兩者是可以共存的。
而且 Hooks 是百分之百向後相容的，所以也不需要考慮到相容性問題。

Hooks 的出現是為了解決 :

  • 在組件間具有狀態的元件很難被重複使用
  • 複雜的 Class 元件變得越難理解
  • Classes 讓人與機器造成混淆

所以在舊有的專案可以逐步轉換成 Hooks 形式，而在新的專案建議直接採用 Hooks 方式開發。

兩個使用 Hooks 的規則 :

  1. Only call Hooks at the top level. Don’t call Hooks inside loops, conditions, or nested functions.
  2. Only call Hooks from React function components. Don’t call Hooks from regular JavaScript functions.
