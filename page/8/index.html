外鍵更新規則

Posted on 2021-11-23 In Database
Symbols count in article: 186 Reading time ≈ 1 mins.

一般外鍵更新的規則有以下四種 :

  1. CASCADE
    只要被參考的欄位被刪除或更新，則對應的外鍵欄位也要一併被刪除或更新。
  2. RESTRICT
    只要有被參考到，則禁止刪除或更新。
  3. NO ACTION
    被參考的欄位被更新或刪除，當作外鍵的欄位不做任何動作，直到根據外鍵做參考時才會發現錯誤。
  4. SET NULL
    被參考的欄位被更新或刪除，則被當作外鍵的欄位將被設定為NULL，而外鍵欄位必需要被允許為空。
